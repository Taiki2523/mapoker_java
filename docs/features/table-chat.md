# テーブルチャット

**ステータス**: `planned`

## 概要

テーブル内のプレイヤーがリアルタイムでメッセージを送受信できるチャット機能を追加する。

## 動機

- プレイヤー同士のコミュニケーションがゲームの楽しさを高める
- 「ナイスハンド」などのリアクションが取れる

## 主要な機能

- テーブルスコープのチャットルーム（テーブル参加者のみ）
- テキストメッセージの送受信
- スタンプ・絵文字リアクション（オプション）

## 技術的な検討事項

- WebSocket（Spring WebSocket / STOMP）か Server-Sent Events（SSE）か
- 現在はポーリングベースのアーキテクチャのため、リアルタイム通信の追加は大きなインフラ変更を伴う
- メッセージの永続化は `chat_messages` テーブルを新規追加する

## 依存する先行作業

- WebSocket 対応（または SSE）インフラの整備
