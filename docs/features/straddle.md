# ストラドル

**ステータス**: `planned`

## 概要

UTG（ビッグブラインドの左隣）のプレイヤーがプリフロップ前に 2BB を強制ポストし、実質的な「3枚目のブラインド」として機能するストラドルを実装する。

## 動機

- ストラドルはアクションを活発にする人気オプション
- 上級者向けのゲーム形式として提供する

## 主要な機能

- テーブル作成時にストラドル有効/無効を選択
- 有効な場合、UTG はゲーム開始前に 2BB を自動ポスト
- ストラドルプレイヤーは最後にアクション権を持つ（BB の左隣から行動開始）
- ストラドル額はプリフロップのミニマムレイズ基準となる

## 技術的な検討事項

- テーブルフラグ（`flags`）に `straddle` を追加、または専用フィールドとして管理
- `startHand()` でストラドルポスト処理を追加
- `currentBet`, `lastRaiseSize` をストラドル額で初期化
- フロントエンドでストラドルポスト表示と UTG への通知
